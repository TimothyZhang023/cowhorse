# Workhorse 产品演进路线图

> 最后更新：2026-03-15  
> 状态：Phase 2 核心能力已完成，进入 Phase 3 准备阶段。

---

## 📅 演进历程与现状 (As-Is)

### ✅ 已完成 (v2.0 桌面优先时代)

-   **核心架构转轨**: 从多租户 Web 应用切换为 `Tauri 2 + Node.js Sidecar` 架构。
-   **单机身份管理**: 移除注册/登录流程，自动注入本地 `local` 用户，支持 SQLite 本地化存储。
-   **完善的 Agent 引擎**: 实现基于 ReAct 循环的任务编排、Token 预算管理与长上下文压缩。
-   **MCP 全力支持**: 完美支持 Stdio 与 SSE 两种 transport，内置 `shell_execute` 安全执行沙箱。
-   **自动化闭环**: 集成 Cron Jobs 定时任务、任务历史审计 trace、钉钉 Webhook 触发。
-   **多端点负载**: 支持 Endpoint 候选集与健康重试机制。

---

## 🚀 未来路线图

### 🟢 Phase 3: 深度连接与协同 (进行中)

-   **[ ] 文件处理增强**: 完善 PDF、Excel 文件的解析能力，支持更大规模的本地文档 RAG。
-   **[ ] 频道集成扩展**: 内置 Telegram、Discord 机器人支持，让 Agent 走出桌面。
-   **[ ] 跨设备联动**: 支持导出配置到其他本地实例，或是通过加密隧道访问其他设备上的 MCP 工具。
-   **[ ] UI 插件系统**: 允许开发者定义自定义卡片，用于展示复杂的工具运行结果（如 3D 渲染、实时图表）。

### 🔵 Phase 4: 自主进化 (规划中)

-   **[ ] 技能自学**: Agent 能够记录用户的手动操作过程，并自动提取为可复用的 `Skill`。
-   **[ ] 向量化记忆**: 将长期的历史对话进行向量化索引，实现跨会话的语义召回。
-   **[ ] 小模型路由**: 针对简单工具调用，自动切换至本地运行的 Llama-3 (通过 Ollama/Llama.cpp) 以节省成本并提升隐私。

## 竞品对标与独特性

| 特性 | Open WebUI | LobeChat | **Workhorse** |
| :--- | :--- | :--- | :--- |
| **部署难度** | 中 (Docker 为主) | 低 (Vercel) | **极低 (单机 .app)** |
| **本地工具调用** | 需要复杂配置 | 依赖浏览器插件 | **原生 Sidecar / MCP** |
| **定时任务** | ❌ (暂无直观入口) | ❌ | **✅ 内置 Cron Runner** |
| **数据主权** | 高 | 中 | **极高 (SQLite 同步写)** |
