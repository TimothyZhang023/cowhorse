# workhorse (CW) — Agent / Cron 运行可观测性升级 Spec

> 最后更新：2026-03-08 ｜ 基于浏览器验收与真实回归日志

## 1. 背景

当前版本已经具备 Agent Tasks、MCP、Cron Jobs 的基础能力，但在真实验收中暴露出几个关键短板：

- 手动启动任务时，缺少运行时输入与调试视角
- 任务启动后，用户很难快速定位到对应会话
- Cron 页面对后台状态变化缺少自动刷新与手动刷新能力
- 长任务在高频 Cron 下会产生重叠执行风险
- 工具调用过程虽然已落库，但页面侧缺少清晰的运行反馈

这些问题不一定会让系统“不能用”，但会明显拉低产品的专业感、可运维性和可解释性，和行业先进产品仍有差距。

## 2. 升级目标

本轮升级聚焦一个目标：

- 把 Agent 与 Cron 从“能执行”升级到“可调试、可观测、可追踪”

用户应当能够：

- 在启动任务前补充本次运行目标
- 在启动成功后直接跳转到对应会话查看全过程
- 在 Cron 页面持续看到最新执行状态，而不是只依赖后台日志
- 避免同一个 Cron 任务在上一次还没结束时被重复拉起

## 3. 范围

### 本轮实现

- `任务编排` 页面新增 `Run & Debug` 启动弹窗
- 任务运行成功后返回会话结果卡片，并支持一键跳转聊天页
- `聊天` 页面支持通过 `conversationId` 深链打开指定会话
- `定时调度` 页面增加自动刷新与手动刷新入口
- `CronRunner` 增加同任务重叠执行保护
- AgentTask 增加重复工具调用预算与强制总结收敛
- 聊天页把工具执行结果展示为结构化卡片，而不是原始协议串
- 保留当前数据库与 API 结构，不引入破坏性 schema 变更

### 本轮不实现

- Tool timeline 独立可视化组件
- Agent 运行中间步骤的专用调试页
- 前端全部 Ant Design 告警一次性清零

## 4. 详细设计

### 4.1 Run & Debug 弹窗

在 `任务编排` 页中，点击 `启动` 不再直接盲调接口，而是先打开一个运行弹窗。

弹窗包含：

- 任务名称
- 当前绑定的技能标签
- 当前绑定的工具标签
- 本次运行目标输入框（可为空）
- 运行结果区域

交互规则：

- 如果用户填写运行目标，则作为 `message` 传给后端
- 如果用户留空，则后端使用兜底 user message
- 运行成功后在弹窗中展示：
  - `conversationId`
  - `finalResponse` 摘要（若为空则提示去聊天页查看工具轨迹）
  - `前往会话` 按钮

### 4.2 Chat 会话深链

聊天页支持读取 URL 查询参数：

- `/chat?conversationId=<id>`

规则：

- 若该会话存在，则优先选中该会话
- 若不存在，则回退到默认首个会话
- 不改变现有会话列表与消息加载逻辑

### 4.3 Cron 页可观测性增强

在 `定时调度` 页面增加两类能力：

- 手动刷新按钮
- 自动刷新（页面可见时每 10 秒拉取一次）

页面需要持续更新：

- `next_run`
- `last_run`
- `last_status`

显示要求：

- `running` 用蓝色状态标签
- `success` 用绿色状态标签
- `failed` 用红色状态标签
- 其他未知状态保持默认样式

### 4.4 Cron 并发保护

在 `CronRunner` 中对单个 job 增加运行锁：

- 同一 job 已在运行时，下一次触发直接跳过
- 输出 warn 级日志
- 不再允许后台出现相同 `jobId` 的重叠 Agent 执行

### 4.5 Agent 收敛与工具结果可读性

验收中已经出现 Agent 连续重复调用同一个工具、直到达到 `MAX_TURNS` 上限且没有 `finalResponse` 的情况。本轮补充两条收敛策略：

- 单个 `tool_name + arguments` 组合增加重复调用预算
- 命中预算或达到最大轮数后，触发一次“禁止继续调用工具”的强制总结

交互与实现规则：

- 同参同工具默认最多执行 2 次
- 第 3 次再请求相同调用时，不再真正执行工具，而是回写一条 budget hit 的 tool message
- 随后立即发起一次无 tools 的总结请求，要求模型基于现有上下文给出最终结论
- 若总结仍为空，则返回降级提示，引导用户去聊天页查看最近一次工具结果

为了让页面更易理解，`/conversations/:id/messages` 需要把历史中的：

- `[TOOL_CALLS]:...`
- `[TOOL_RESULT:<id>:<name>]:...`

转换成更适合前端直接渲染的结构化消息字段，而不是把协议串原样暴露给 UI。

## 5. 验收标准

- `任务编排` 页点击 `启动` 会出现调试弹窗
- 弹窗内支持输入运行目标并发起执行
- 执行成功后可以一键跳转到对应聊天会话
- 聊天页通过 `conversationId` 深链能打开指定会话
- `定时调度` 页支持手动刷新
- `定时调度` 页在不手动操作的情况下也会刷新执行状态
- 高于任务执行耗时的 Cron 频率不再产生重叠执行
- AgentTask 在重复工具调用或达到轮数上限时，仍能返回可展示的最终总结或明确降级提示
- 聊天页不再直接展示 `[TOOL_RESULT:...]` 原始协议串

## 6. 后续建议

下一轮建议继续推进：

- Agent 工具调用预算、终止条件与最终总结阶段
- Task 运行 Timeline / Event Stream 面板
- Cron 执行历史表与耗时统计
- 统一迁移 Ant Design deprecated API 与 message/modal 上下文用法
